# 绕桩小车 —— 综合实验项目文档

> **项目**：电子系统导论综合实验（魔方绕桩）  
> **最后更新**：2026-06-01

---

## 一、赛道规则

| 魔方颜色 | 通过规则 | 动作要点 |
|:---|:---|:---|
| **黄色** | 从魔方**左侧**通过 | 左转切向 → 左侧侧身直行 → 右转回正 |
| **红色** | 从魔方**右侧逆时针绕行 360°** | 摄像头对准 → 直线接近入轨 → 原地右转90°切向对齐 → 弧线绕行360° → 回正 |
| **绿色** | 从魔方**右侧**通过 | 右转切向 → 右侧侧身直行 → 左转回正 |

- 起点静止出发，全程**禁止人工干预**
- 限时 **1分钟**，超时部分不计成绩
- 3个魔方在赛道中间**随机位置**摆放，顺序不固定

### 里程碑评分
| 档位 | 条件 |
|:---|:---|
| 0档 | 完全不能起步 |
| 1档 | 起步但未过第一里程碑 |
| 2档 | 过了第一但未完成第二 |
| 3档 | 完成第二但未通过第三 |
| 4档 | 通过第三但未到达终点 |
| 5档 | 顺利到达终点 |

## 二、硬件配置与接线

### 2.1 电机驱动（L298N）

| 树莓派GPIO(BCM) | 连接对象 | 功能 | 电平说明 |
|:---|:---|:---|:---|
| 13 | ENA | 右电机PWM使能 | PWM 1000Hz |
| 26 | IN1 | 右电机方向A | **高电平 = 右轮前进** |
| 19 | IN2 | 右电机方向B | 高电平 = 右轮后退（制动用） |
| 16 | ENB | 左电机PWM使能 | PWM 1000Hz |
| 21 | IN3 | 左电机方向A | 高电平 = 左轮后退（制动用） |
| 20 | IN4 | 左电机方向B | **高电平 = 左轮前进** |
| 6  | LS  | 左编码器（霍尔传感器） | 上升沿中断计数 |
| 12 | RS  | 右编码器（霍尔传感器） | 上升沿中断计数 |

> **注意**：左右电机前进方向定义不同（右轮IN1高，左轮IN4高），`motor.py` 已封装 `_set_right()` / `_set_left()` 统一为正占空比前进。

### 2.2 外接传感器

| 传感器 | 接口 | 用途 | 关键配置 |
|:---|:---|:---|:---|
| **摄像头** | USB (`/dev/video0`) | 颜色识别（黄/红/绿）+ 辅助验证绕行完成 | 320×240@30fps，ROI下半部 |
| **KS103超声波** | I2C (bus=1, addr=0x74) | 精确测距 + 与面积融合校验 | 模式 `0xb0`，35ms读取间隔 |

## 三、系统架构

```
┌─────────────────────────────────────────┐
│  感知层 (Sensors)                        │
│  • camera.py   → OpenCV颜色识别 (HSV+轮廓)│
│  • ks103.py    → I2C超声波 + 中值滤波     │
│  • encoder.py  → 霍尔中断脉冲计数         │
├─────────────────────────────────────────┤
│  决策层 (Navigation)                     │
│  • state_machine.py → 主状态机 (10Hz tick)│
│  • maneuvers.py     → 黄/绿机动动作序列   │
├─────────────────────────────────────────┤
│  控制层 (Control)                        │
│  • pid.py      → 增量式PID (微分先行)     │
│  • motor.py    → GPIO+PWM + PID速度环线程 │
└─────────────────────────────────────────┘
```

### 核心设计约束
- **禁止轮子倒转**：所有转弯采用比例差速（`turn_bias`），两轮始终正转，避免原地打转卡死。
- **低分辨率低帧率**：320×240 + 10~20Hz 主循环，适配老旧树莓派算力。
- **颜色指定检测**：`detect_cube()` 只检测单一目标颜色，抗干扰；`identify_nearest_cube()` 仅在近距离确认颜色时遍历三色。

## 四、状态机设计（核心逻辑）

### 4.1 黄/绿魔方通过流程

每个魔方拆分为 **3 个机动步骤**，由 `maneuvers.py` 定义，`state_machine.py` 顺序消费：

```
SEARCH  →  直线搜索（PID匀速，摄像头扫色）
    ↓ (摄像头发现颜色 且 超声波 < detect_dist)
ALIGN   →  切向转向（黄=左转，绿=右转，定时/超声触发退出）
    ↓ (超时 或 超声波 < action_dist)
PASS    →  侧身直行（轻微同向偏置保持间距，超声"先减后增"退出）
    ↓ (超时 或 超声脱离特征)
RECOVER →  回正车身（反向差速恢复直线）
    ↓ (超时)
SEARCH  →  进入下一个魔方搜索（或 FINISH）
```

### 4.2 红色魔方 360° 绕行流程（切向入轨策略）

```
SEARCH
    ↓ (发现红色 且 超声波 < detect_dist)
ALIGN_RED  →  摄像头P控制器对准魔方中心（cx误差 → 转向偏置）
    ↓ (超声波 < orbit_radius + approach_margin + 10cm)
APPROACH   →  直线前进，直到距离 ≤ orbit_radius（入轨半径）
    ↓ (到达入轨距离 或 超时)
TANGENT    →  原地差速右转90°（左轮快、右轮慢，两轮均正转）
    ↓ (编码器累计右转角度 ≥ 90° 或 超时)
ORBIT      →  固定差速弧线绕行360°
    ↓ (编码器累计右转 ≥ 360°，可选摄像头二次确认 或 超时)
RECOVER_RED→  轻微左转回正
    ↓ (超时)
SEARCH / FINISH
```

**关键参数**（见 `config.py`）：
- `orbit_radius = 25` cm：小车中心到魔方中心的绕行半径
- `tangent_target_deg = 90`°：原地右转目标角度
- `deg_per_pulse_diff`：编码器转角标定系数（**必须用 `cal_angle.py` 实车标定**）

## 五、控制策略与参数

### 5.1 PID速度环（增量式 + 微分先行）

```python
# control/pid.py
# 微分先行：对 setpoint 跳变免疫，适合每周期动态调整目标速度
L = PIDController(Kp=25, Ki=0.01, Kd=35, ideal_speed=0.61)
R = PIDController(Kp=25, Ki=0.01, Kd=40, ideal_speed=0.57)
# 采样周期 dt 隐含在 Ki/Kd 中，由 config.MOTOR['pid_period']=0.1s 定频调用
```

### 5.2 差速转向（不倒转设计）

```python
# control/motor.py
left  = speed * (1.0 + turn_bias * turn_rate)
right = speed * (1.0 - turn_bias * turn_rate)
# 硬限幅：max(min_duty, min(100, duty))，确保两轮始终正转
```

### 5.3 典型机动参数（config.py）

| 动作 | 速度(占空比) | 转向偏置 | 持续时间/退出条件 | 说明 |
|:---|:---|:---|:---|:---|
| 直线搜索 | 40 | 0 | 持续 | PID闭环 |
| 黄/绿 ALIGN | 35 | ±0.7 | 0.4s 或 超声<30cm | 符号由颜色决定 |
| 黄/绿 PASS | 30 | ±0.2 | 1.0s 或 超声"先减后增" | 保持侧身间距 |
| 黄/绿 RECOVER | 35 | ∓0.6 | 0.4s | 与ALIGN反向 |
| 红 ALIGN_RED | 30 | 动态(P控制) | 6s 或 超声<40cm | cx误差×0.4 |
| 红 APPROACH | 30 | 0 | 5s 或 超声≤25cm | 直线入轨 |
| 红 TANGENT | 18 | +0.85 | 3s 或 编码器≥90° | 原地右转切向 |
| 红 ORBIT | 25 | +0.5 | 8s 或 编码器≥360° | 弧线绕行 |
| 红 RECOVER_RED | 35 | -0.6 | 0.4s | 左转回正 |

## 六、传感器融合策略

### 6.1 分层职责

| 传感器 | 主任务 | 辅助任务 |
|:---|:---|:---|
| **摄像头** | 远距离(>50cm)颜色识别，决定进入哪种机动序列 | 红色ORBIT完成后二次确认 |
| **KS103超声波** | 精确测距(20-60cm)，触发各阶段退出 | 黄/绿PASS阶段"先减后增"脱离判定 |
| **编码器** | PID速度闭环 | 红色TANGENT/ORBIT角度累计 |

### 6.2 KS103 面积-超声融合校验

外组因 KS103 数据跳变和锥形波束误检，最终完全放弃超声波改用图像面积估算距离。**本组策略不同**：

- **面积不直接换算距离**（受魔方朝向、俯仰角、遮挡、光照影响极大，难以精确标定）。
- **超声波为主、面积为辅**：面积作为超声波读数的 **"趋势校验器"**。

**融合逻辑**（`sensors/ks103.py`）：
- 当超声波新读数与历史 median 偏差 > 15cm 时，进入面积趋势校验：
  - **面积增大（远离）但超声骤减** → 超声检测到地面/侧方反射（锥形波束），**丢弃**
  - **面积减小（靠近）但超声骤增** → 超声丢波/超量程，**丢弃**
  - **趋势一致** → 接受读数，可能只是小车快速机动
- 这样既保留超声波绝对精度，又利用面积趋势稳定性弥补跳变缺陷。

## 七、开发工具与调试

### 7.1 HSV 颜色阈值实时校准 (`tools/calibrate_hsv.py`)

```bash
python3 tools/calibrate_hsv.py
```

- 启动滑条窗口，实时调节 H/S/V 上下界
- 按 `r/y/g` 切换颜色，`1/2` 切换红色双区间
- 按 `s` 保存到 `calibration_hsv.json`，后续 `camera.py` 自动加载
- 按 `q/ESC` 退出

> 赛道光照与实验室差异大，比赛前必须用此工具重新标定阈值。

### 7.2 编码器转角系数标定 (`tools/cal_angle.py`)

```bash
python3 tools/cal_angle.py
```

- 按回车启动，小车以正式比赛参数（`orbit_speed=25, turn_bias=0.5`）绕魔方
- 肉眼观察小车回到魔方右侧时按 `Ctrl+C`
- 自动计算 `K = 360° / 脉冲差`，填入 `config.MANEUVER['deg_per_pulse_diff']`

> **标定准确性直接决定红色360°绕行成败**，默认 `0.2` 大概率不准，必须实测。

### 7.3 单模块自检入口

每个驱动文件均提供 `if __name__ == '__main__':` 自检：

| 文件 | 自检内容 | 运行命令 |
|:---|:---|:---|
| `control/motor.py` | 双轮低速前进2秒，打印编码器脉冲数 | `python3 control/motor.py` |
| `sensors/ks103.py` | 连续测距，显示滤波距离与原始历史 | `sudo pigpiod && python3 sensors/ks103.py` |
| `sensors/camera.py` | 实时颜色识别可视化，按 r/y/g 切换 | `python3 sensors/camera.py` |

## 九、已知问题与修复记录

| 问题 | 状态 | 修复说明 |
|:---|:---|:---|
| 差速过大会导致轮子倒转/停转，小车原地打转 | ✅ 已修复 | `motor.py` 使用比例差速 + 硬限幅 `[min_duty, 100]`，确保两轮始终正转 |
| PID参数过高导致振荡 | ✅ 已修复 | 实验7教训 `(90,60,100)` 弃用，当前采用保守低速参数 `(25,0.01,35/40)` |
| 左右电机物理不对称 | ✅ 已修复 | 差异化目标速度 (`0.61` vs `0.57`) + 差异化 Kd |
| 红色魔方定时硬编码不可靠 | ✅ 已修复 | 改用编码器角度累计 + 超声波距离融合判定 |
| `motor.py` 制动方向与前进相同（无效制动） | ✅ 已修复 | 左轮制动引脚方向修正：`IN3=HIGH, IN4=LOW`（与前进相反） |
| 状态机 `_advance()` 单tick跳过PASS/RECOVER | ✅ 已修复 | 改为 `_complete_step()`，每次只推进一步，按 phase 切换状态 |
| KS103 数据跳变/锥形波束误检 | ✅ 已缓解 | 中值滤波 + 面积趋势校验，而非完全弃用超声 |

## 十、关键参数修改点

| 场景 | 修改文件 | 关键参数 |
|:---|:---|:---|
| 电机不转/抖动 | `config.py` | `MOTOR['min_duty']`（默认5，可提高到8-10） |
| 转弯半径不对 | `config.py` | `MANEUVER['*_turn']` 或 `turn_rate` |
| 红色绕行角度不准 | `config.py` | `MANEUVER['deg_per_pulse_diff']`（必须标定） |
| 颜色识别失败 | `tools/calibrate_hsv.py` | 实时调节并保存 `calibration_hsv.json` |
| 超声数据跳变 | `config.py` | `KS103['outlier_threshold_cm']`、`filter_window` |
| 动作超时/不够 | `config.py` | `MANEUVER['*_time']` 或 `*_timeout` |
