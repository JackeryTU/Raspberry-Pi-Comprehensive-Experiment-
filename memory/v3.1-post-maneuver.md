# v3.1 完成后扫描与 PD 对准

## 动机

**问题**：v3 完成一个魔方后直接跳转 SEARCH，小车可能已偏离下一魔方朝向，导致 SEARCH 需要较长时间才能重新发现目标。

**方案**：在每个魔方通过完成后，插入三段流水线：等待 → 原地旋转扫描 → PD 对准，使小车主动寻找并对准下一个魔方，然后交给 SEARCH 快速锁定路由。

## 新增状态流转

```
MANEUVER/RECOVER_RED → POST_WAIT → SCAN_NEXT → ALIGN_CUBE → SEARCH → (下一魔方)
```

### POST_WAIT
- 停车等待 `post_maneuver_wait` 秒（默认 0.5s），消除前序机动惯性
- 超时后自动进入 SCAN_NEXT

### SCAN_NEXT（v3.10 更新）
- 两阶段原地旋转扫描，纯摄像头检测，**有超时保护**：
  1. `left` — 左转 `scan_next_angle`°（默认 50°），超时 `scan_next_timeout`（6s）
  2. `right` — 右转 `scan_next_angle × 2`°（默认 100°），超时 `scan_next_timeout × 2`（12s）
- 任一阶段摄像头检测到魔方 → 记录颜色到 `_next_color`，进入 ALIGN_CUBE
- 两阶段均未检测到 → 回退 SEARCH（保底）
- **v3.10 修复**：阶段切换前先 `set_target(0,0)` 停车，防止上一阶段方向惯性残留

### ALIGN_CUBE（v3.10 更新）
- **PD 摄像头比例对准**：
  ```
  err = (cx - w/2) / (w/2)        # 归一化偏差 [-1, +1]
  derr = err - prev_err           # 微分项
  turn_power = err × kp + derr × kd  # PD 控制
  ```
- 阶段 1 `rotate`：原地左右转，PD 控制对准摄像头中心，超时 `align_cube_rotate_timeout`（4s）
- 阶段 2 `forward`：对准后直线前进到停止距离
- **退出条件**：
  1. 超声波距离 ≤ `align_cube_stop_dist`（默认 35cm）→ 进入 SEARCH
  2. 超时 `align_cube_timeout`（默认 8s）→ 保底进入 SEARCH

## 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `post_maneuver_wait` | 0.5 | 完成后等待时间（秒） |
| `scan_next_speed` | 20 | 扫描旋转速度（占空比），v3.10 从 15 提高 |
| `scan_next_angle` | 50 | 单侧扫描角度（度） |
| `scan_next_timeout` | 6.0 | 单次扫描超时（秒），v3.10 新增 |
| `align_cube_kp` | 0.25 | 对准 P 增益 |
| `align_cube_kd` | 0.05 | 对准 D 增益 |
| `align_cube_deadband` | 0.05 | 对准死区（归一化偏差） |
| `align_cube_rotate_speed` | 20 | 原地旋转对准的最大速度 |
| `align_cube_rotate_timeout` | 4.0 | 旋转对准阶段超时（秒） |
| `align_cube_speed` | 26 | 对准前进速度（占空比） |
| `align_cube_timeout` | 8.0 | 全局超时（秒） |
| `align_cube_stop_dist` | 35 | 对准停止距离（cm） |

## v3.10 关键修复

### 扫描超时保护
SCAN_NEXT 的 left/right 阶段仅靠编码器角度判定退出。若 `scan_next_speed` 太低导致电机堵转，编码器脉冲永不增长，小车永久卡死。v3.10 添加 `scan_next_timeout` 超时保护，超时后强制推进。

### 阶段切换停车
SCAN_NEXT 从左扫切换到右扫时，上一阶段的 `set_raw` 指令继续跑一个 tick（~67ms），导致方向惯性残留。v3.10 在阶段切换前先 `set_target(0,0)` 停车。

### 扫描速度提高
`scan_next_speed` 从 15 提高到 20，原地旋转需要一正一反克服双倍静摩擦，15% 偏临界。

## 影响范围

- `config.py` — 新增 `scan_next_timeout`，`scan_next_speed` 15→20
- `navigation/state_machine.py` — SCAN_NEXT 阶段加超时保护 + 切换前停车
- 黄/绿/红机动逻辑不变

**Why:** 主动扫描对准比被动直行搜索更可靠，减少遗漏概率；超时保护防堵转卡死。
**How to apply:** 无需修改 maneuvers.py；所有新参数在 config.py 的 MANEUVER 段直接调节。
